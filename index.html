AI-оператор Google Ads · Rust

Ничего не тратится
без сигнала «можно».

ads-ai анализирует кампании, предлагает каждое изменение бюджета, паузу, минус-слово — и не применяет ни одного без явного человеческого одобрения. Автономность — это не отмена правила, а заранее подписанный пропуск для узкого класса безопасных решений.

Компонентная схема

Один оркестратор, два мокаемых контура, один шлагбаум

CLI и демон — один и тот же код. Обе внешние зависимости — Claude и Google Ads — спрятаны за enum-диспетчером с полноценным моком, поэтому весь цикл прогоняется без единого ключа. На всей схеме только одна стрелка умеет тратить деньги.

живой вызов
не подключено или opt-in
требует пропуска через gate
D-008 · ревизия D-002

Автономность — это подписанный пропуск, не отмена контроля

По умолчанию агент только предлагает. autonomy.enabled = true открывает второй, узкий вход в тот же самый шлагбаум — не обходной путь.

Вход 1 · по умолчанию

Человек — proposals approve

Вы просматриваете proposals list / show <id> и одобряете вручную, по одному или разом через --all.

Вход 2 · opt-in

Политика — [autonomy] в конфиге

Заранее сконфигурированное предразрешение для узкого класса действий, применяемое внутри analyze_once() сразу после генерации предложения.
